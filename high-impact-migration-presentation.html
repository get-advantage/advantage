One Important Change: Configuration Pattern

Moving from bundled configs to separate config files

üì¶ Old: High Impact JS

Bundled Configuration

Each publisher gets a custom-built JavaScript bundle with their site-specific settings compiled into the code.

// Custom bundle for Publisher A highimpact-publisherA.min.js // Custom bundle for Publisher B highimpact-publisherB.min.js
  • New bundle needed for each site
‚öôÔ∏è New: Advantage

Separate Configuration

One universal bundle for everyone. Each publisher has their own config file with site-specific settings.

// Universal bundle advantage.js // + Publisher's config publisher-config.js
  • One core bundle for all sites
  • Config updates without redeployment of core bundle
  • Can be hosted by publisher or remotely

Migrating Configuration to Advantage Style

The Migration Path

1 Extract Settings from Bundle

Your integration team identifies the site-specific settings currently compiled into your High Impact JS bundle (z-indexes, element hiding, responsive adjustments, etc.)

2 Create Advantage Config File

Convert these settings into an Advantage formatIntegrations config file. This uses simple JavaScript with setup/close/reset functions for each format.

3 Load Config with Bundle

Either bundle the config with Advantage or load it separately from your server or a 3rd party CDN.

üì¶ Option A: Bundled

Create a custom bundle that includes your config (like advantage-with-your-site-integration.js)

  • One file to load
  • Similar to current approach
  • No extra HTTP request
üåê Option B: Separate (Recommended)

Load universal Advantage bundle + separate config file from your CDN or 3rd party

  • Update settings without redeployment
  • Can be managed by 3rd party
  • More flexible

Technical Details (For the Curious)

What's Inside the Bundle?

üéØ Advantage Core

Modern format system with TopScroll, Midscroll, DoubleMidscroll, and more. Built with Web Components for encapsulation.

üîå Compatibility Layer

Translates High Impact JS API calls into Advantage format commands. Handles GAM integration, message passing, and ad detection.

üé® Responsive Styling

Injects platform-specific CSS into ad iframes (GAM, Adform, Xandr, etc.) to ensure proper banner scaling across all ad servers.

Key Innovations

  • Pre-wrapping: Ad slots are wrapped before ads load, preventing layout shifts
  • Message interception: Listens for format requests from creatives
  • Two-phase rendering: Waits for both ad load and creative signal
  • Platform detection: Automatically applies correct styling per ad server
